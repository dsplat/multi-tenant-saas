## 给 AI 的补充说明

### 开发原则
1. **向后兼容**：所有新增功能必须向后兼容，不能破坏现有API
2. **模块化设计**：所有新功能必须以模块形式实现，支持按需加载
3. **租户隔离**：所有新功能必须支持租户隔离，数据不能跨租户泄露
4. **配置驱动**：所有新功能必须支持配置驱动，不能硬编码
5. **测试覆盖**：所有新功能必须有对应的单元测试和集成测试
6. **文档完整**：所有新功能必须有完整的文档说明

### 技术规范
1. **命名规范**：遵循PSR-12编码规范
2. **类型声明**：所有方法必须有完整的类型声明
3. **PHPDoc**：所有公共方法必须有完整的PHPDoc注释
4. **异常处理**：所有异常必须使用框架定义的异常类
5. **日志记录**：所有关键操作必须有日志记录
6. **性能考虑**：所有新功能必须考虑性能影响

### 代码组织
1. **服务层**：所有业务逻辑必须放在服务层
2. **控制器**：控制器只负责请求验证和响应格式化
3. **模型**：模型只负责数据定义和关联关系
4. **中间件**：中间件只负责请求预处理和响应后处理
5. **配置**：所有可配置项必须放在config目录

### 测试要求
1. **单元测试**：测试单个方法的功能
2. **集成测试**：测试多个组件的协作
3. **功能测试**：测试完整的业务流程
4. **性能测试**：测试关键路径的性能

### 文档要求
1. **完整性**：文档必须覆盖所有功能
2. **准确性**：文档必须与代码保持同步
3. **易读性**：文档必须易于理解
4. **示例性**：文档必须包含示例代码
